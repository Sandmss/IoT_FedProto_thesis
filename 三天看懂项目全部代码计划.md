# 三天看懂项目全部代码计划

## 1. 计划目标

这份计划的目标不是“把每个文件都扫一遍”，而是让你在 3 天内真正建立对整个项目代码的完整理解，最终达到以下标准：

1. 能说清项目的整体功能和运行链路。
2. 能独立解释 `Local`、`FedAvg`、`FedProto` 三种算法在代码里的实现差异。
3. 能说清数据如何进入系统、模型如何训练、结果如何保存、界面如何调度。
4. 能快速定位任何一个核心功能对应的文件。
5. 能为论文中的“系统实现”和“实验设计”部分提供准确代码依据。

## 2. 代码范围划分

当前项目真正需要重点读懂的代码，主要分为 3 层。

## 3. 推荐理解顺序

千万不要按文件名字母顺序硬读，建议严格按下面顺序：

1. 先理解项目整体运行入口
2. 再理解联邦训练主循环
3. 再区分客户端与服务端职责
4. 再理解模型定义
5. 再理解数据加载和结果保存
6. 最后补读脚本和桌面界面

原因：

- 入口不懂，后面每个类都像孤岛
- 主循环不懂，客户端和服务端就只是一堆函数
- 先读桌面界面容易分散注意力

## 4. 三天总策略

三天节奏建议如下：

- 第 1 天：看懂“系统怎么跑起来”
- 第 2 天：看懂“核心算法怎么实现”
- 第 3 天：看懂“辅助工具、结果链路和桌面界面”

每一天都必须输出你自己的阅读产物，而不是只读不记。

建议每天固定产出：

1. 一张模块关系图
2. 一份文件功能清单
3. 一份流程笔记
4. 一组你自己整理的问题与答案
