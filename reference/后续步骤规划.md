# IoT 恶意流量检测 · 联邦原型仿真系统 — 后续步骤规划

本文档在 [`项目与开题差距清单.md`](项目与开题差距清单.md) 与开题报告第四部分的基础上，整理**可执行、可验收**的后续工作。默认目标：在答辩前形成「算法闭环 + 论文级实验 + 可复现仿真系统」三位一体交付物。

---

## 一、总目标与验收口径

| 维度 | 目标（答辩时可陈述） |
|------|----------------------|
| **仿真系统** | 单机多客户端联邦流程可一键复现；配置与结果目录规范；README 或脚本说明「如何从零跑通」 |
| **与开题对齐** | FedProto 主链路、Non-IID 数据、轻量化骨干（MLP / 1D-CNN）、异构场景至少一组对比实验 |
| **论文论证** | 安全检测指标（含不平衡任务常用指标）+ 与 FedAvg / Local 基线对照 + 通信/算力侧量化（至少通信与参数量） |

建议在仓库中固定一个 **`experiments/` 或 `configs/`** 目录存放 YAML/JSON 与批处理脚本，避免参数散落在 shell 里难以复现。

## 二、阶段划分（推荐顺序）

### 阶段 0：稳定性与可复现基线（约 3～5 天）

**目的**：任何改动前，保证「FedProto / FedAvg / Local」在目标数据集与默认超参下能稳定跑完并产出结果文件。

| 步骤 | 工作内容 | 产出 / 验收 |
|------|----------|-------------|
| 0.1 | 固定一套**标准命令**（数据集路径、`input_dim`、`num_classes`、轮次、随机种子），写入 `README` 或 `docs/运行说明.md` | 新成员按文档可跑通 |
| 0.2 | 跑通 **FedProto** 全流程（含 `--skip_figures` 与不含），确认训练结束无未捕获异常 | 日志 + `results/*.h5` |
| 0.3 | 同样配置跑 **FedAvg**、**Local**，保存到不同 `goal` 或 `save_folder_name`，避免覆盖 | 三套 h5 或等价结果 |
| 0.4 | 记录当前最佳 Acc / AUC 区间与大致训练时间，作为后续对比的**基线表 v0** | 一页表格（可放论文附录） |

**注意**：Windows 下以 `python src/main.py ...` 为准；`.sh` 可在 WSL/Git Bash 使用或改为 `.ps1`。

---

### 阶段 1：评估体系与「全局测试」定义（约 5～8 天）

**目的**：满足开题第四部分（四）中「混淆矩阵、精确率、召回率、综合指标」的论证强度；并与开题「预留全局测试集」表述一致或主动收窄表述。

| 步骤 | 工作内容 | 产出 / 验收 |
|------|----------|-------------|
| 1.1 | **明确评估对象**（二选一并在论文中写死）：<br>• **方案 A**：保留现有「各客户端本地测试集 + 按样本数加权平均」；开题中「全局测试集」改为「联邦标准下的全局性能估计」。<br>• **方案 B**：从原始划分中预留**一份不参与训练的中央测试集**（如 `dataset/IoT/global_test`），每轮仅在服务器侧评估一次。 | 论文「实验设置」一节文字 + 目录结构 |
| 1.2 | 在 `evaluate` 路径上增加 **Precision / Recall / Macro-F1**（宏平均适合类别不平衡）；可选 Weighted-F1 | 每轮打印 + 写入 h5 |
| 1.3 | 每 `eval_gap` 轮或最后一轮保存 **混淆矩阵**（numpy + `matplotlib` 热图或仅 csv） | `results/figures/` 或 `results/metrics/` |
| 1.4 | 扩展 `utils/result_utils.py`：读取新字段、多次 run 的 mean/std | 与现有 `average_data` 风格一致 |

**验收**：同一配置下，FedProto 与 FedAvg 至少各跑一次，**同一张对比表**含 Acc、Macro-F1、Macro-P/R（及 AUC 若保留）。

---

### 阶段 2：异构客户端（开题核心卖点）（约 5～10 天）

**目的**：实现开题（二）2 中「网关用 1D-CNN、低端节点用 MLP」的**同一联邦轮次内**异构，而非仅两次独立实验切换 `model_family`。

| 步骤 | 工作内容 | 产出 / 验收 |
|------|----------|-------------|
| 2.1 | 设计配置项：如 `hetero_mode=off|split|by_list`，`cnn_client_ratio` 或 `cnn_client_ids=[0,1,5,...]` | `args` + 配置文件示例 |
| 2.2 | 修改客户端创建逻辑（`Server.set_clients` 或 `main.py` 传入 per-client 模型描述）：保证 **最后一维 `feature_dim` 一致**，骨干不同 | 单元测试或启动时 assert |
| 2.3 | 确认 FedProto 聚合仍按 **类样本数加权**（与架构无关）；本地训练与原型维度一致 | 异构下无 shape 错误 |
| 2.4 | 做**至少两组**实验：全 MLP、全 CNN、异构混合（固定 Non-IID 与随机种子） | 论文表 + 曲线 |

**验收**：论文中可写「部分客户端 MLP、部分 CNN1D，服务端仅聚合原型」，并给出精度与通信对比（通信可在阶段 3 补数）。

---

### 阶段 3：轻量化与通信 / 计算量化（约 5～8 天）

**目的**：支撑开题「双重轻量化」与「与 FedAvg 通信量对比」的定量叙述。

| 步骤 | 工作内容 | 产出 / 验收 |
|------|----------|-------------|
| 3.1 | **通信量（仿真）**：FedProto 每轮上传/下载「原型张量」元素个数 × dtype 字节；FedAvg 每轮上传/下载「全模型参数」字节数；累计曲线或表 | `results/comm_stats.json` 或写入 h5 |
| 3.2 | **参数量**：对每个 `model_family`（及异构组合）统计 `sum(p.numel())` | 论文表一行 |
| 3.3 | **推理延迟**：单进程对单 batch 或固定 N 条样本测 wall-clock（CPU/GPU 注明）；可选 `fvcore` / thop 估计 **FLOPs** | 表格：MLP vs CNN vs 异构 |
| 3.4 | 用 **精度–通信–延迟** 画一张二维或雷达图（论文用） | `results/figures/` |

**验收**：开题中「指数级下降」改为数据支撑表述（附具体倍数，避免空洞）。

---

### 阶段 4：剪枝 / 量化（可选，按时间裁剪）（约 5～10 天或降级为「未来工作」）

**目的**：对应开题「预留剪枝或量化接口」；若时间不足，可在论文中写「本文完成接口与一种设置下的初步实验」。

| 步骤 | 工作内容 | 产出 / 验收 |
|------|----------|-------------|
| 4.1 | 二选一优先：**训练后剪枝 + 微调** 或 **动态量化（`torch.quantization` 推理）** | 开关参数 `--prune` / `--quantize` |
| 4.2 | 报告压缩率 vs Acc / Macro-F1 变化 | 小消融表 |
| 4.3 | 若与 FedProto 叠加，说明「本地压缩 + 原型通信」两条线各自收益 | 论文一段讨论 |

**若时间不足**：阶段 4 整体标为「展望」，阶段 3 仍足以支撑「轻量化」主线。

## 四、风险与依赖

- **数据路径与 `num_classes` / `input_dim`**：任何新划分或新数据集需同步脚本与 `main.py` 参数，建议在配置文件中单一真源（single source of truth）。
- **异构与 FedAvg**：FedAvg 要求同构；异构实验表中应对 FedAvg 标注「仅同构对照」或单独「分组 FedAvg」，避免方法学被质疑。
- **GPU 与 Windows**：推理延迟、FLOPs 统计需在论文中注明硬件与 batch 设置。
