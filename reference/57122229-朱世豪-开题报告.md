# 一、选题背景和意义：

# 课题的研究背景：

随着 5G 通信和万物互联技术的迅猛发展，物联网（IoT）设备在智能家居、工业控制、智慧城市等领域的部署规模呈现出爆炸式增长。然而，由于物联网底层设备通常受限于计算、存储和电池资源，难以部署复杂的安全防护机制，使其成为了网络攻击的重灾区。近年来，以 Mirai、BASHLITE 为代表的僵尸网络病毒肆虐，它们通过感染海量脆弱的物联网终端，组建庞大的僵尸网络，进而发起大规模的分布式拒绝服务（DDoS）等恶意流量攻击，对网络空间的安全与稳定造成了极其严重的威胁。

面对严峻的物联网安全态势，传统的恶意流量检测系统多采用集中式的深度学习分析范式。这种范式要求将分布在各个边缘节点的海量流量数据实时上传至云端数据中心进行集中训练与检测。然而，在实际的物联网场景中，该方法面临着难以克服的挑战。首先，物联网设备分布极其广泛且网络带宽往往受限，海量数据的持续上传会造成巨大的通信网络负担，甚至导致严重的检测延迟。其次，也是更为致命的一点，物联网网络流量中往往蕴含着大量高度敏感的用户行为特征与业务隐私数据。随着全球数据安全与隐私保护法规的日益完善，集中式收集和处理底层流量数据面临着严峻的数据泄露风险与合规性挑战，导致不同机构与设备间形成了难以打破的“数据孤岛”。

为了在打破“数据孤岛”的同时兼顾隐私保护，联邦学习作为一种“数据不动模型动”的分布式机器学习新范式，被广泛引入到物联网恶意流量分析领域。联邦学习允许各参与节点在本地利用私有数据训练检测模型，仅将模型更新上传至中心服务器进行聚合，从而在从根本上阻断原始流量数据泄露路径的前提下，实现全局模型的联合优化。

然而，以传统联邦学习框架在向真实的物联网环境落地时，依然暴露出明显的局限性：一方面，深度学习模型动辄包含数以百万计的参数，频繁地在端云之间传输庞大的模型参数会产生极大的通信开销，极易耗尽窄带物联网宝贵的通信资源；另一方面，传统联邦学习强制要求所有参与节点必须运行完全相同架构的同构模型，这严重脱离了物联网设备算力参差不齐（如边缘智能网关与底层微型传感器算力悬殊）的“硬件异构”客观现实。

为了突破传统“参数聚合”模式的理论瓶颈，学术界开始探索更加轻量、解耦的分布式通信新范式。其中，“原型学习”因其高度抽象的特性进入了安全研究视野。因此，如何在保障物联网流量数据绝对隐私的前提下，利用低维度的“特征原型”替代庞大的模型参数进行端云交互，突破通信带宽受限与设备硬件异构的双重瓶颈，构建出一种既“轻量”又“兼容异构”的分布式协同防御机制，成为了当前物联网安全领域亟待解决的关键难题。这也构成了本课题研究的核心出发点。

# 课题的研究意义：

1. 将联邦原型学习引入物联网安全领域，探索在数据非独立同分布和硬件异构场景下的恶意流量检测新范式。通过研究基于一维流量统计特征的原型提取与聚合机制，为边缘网络中的模型知识共享提供了新的理论视角。

2. 本课题针对物联网场景设计了轻量化恶意流量检测方案。具体来说，一方面采用轻量化特征提取模型，大幅降低了底层设备的计算负担；另一方面，通过传输低维度的原型向量代替全量模型参数，通信开销呈指数级下降，完美契合窄带物联网环境。该系统在贯彻“数据不出本地”原则的同时，能够灵活、安全地部署于各类算力受限的终端，显著提升物联网的主动防御能力。

3. 构建端云协同的闭环恶意流量检测系统。本课题将理论算法转化为具体的软件代码，实现了涵盖“本地流量特征提取、端云原型交互、云端全局聚合与本地模型微调”的完整运行流程。该系统的实现使得原本孤立的物联网设备能够协同共享恶意流量特征，验证了基于轻量化联邦学习的协同防御方案在实际工程落地中的可行性。

# 二、课题关键问题及难点：

# 关键问题：

1. 轻量化恶意流量特征提取模型的设计

物联网设备的计算与存储资源极度受限。因此，应采用低参数量与计算复杂度的本地检测模型。关键在于从原始网络流量数据中解析出有效特征，构建轻量化特征提取网络，完成对网络流量数据的特征提取。

2. 基于原型聚合的高效联邦学习框架构建

打破传统联邦学习传输全量模型参数的模式，引入联邦原型学习（FedProto）机制。“特征原型”本质上是某一类恶意流量数据在神经网络隐空间中的特征均值向量，它能够高度概括该类流量的本质属性且维度极低。 本课题的关键工作正是基于这一概念，实现底层网络架构解耦的协同训练。通过允许各个物联网节点运行不同结构的本地模型，并在服务端仅通过聚合各类流量的“特征原型”来更新全局知识，从而在保证物联网流量数据隐私且不额外增加通信负担的前提下，实现高效的恶意流量联合检测。

# 课题难点：

1. 高动态流量数据向“特征原型”的模态适配与表征

现有的 FedProto 算法主要针对标准的二维图像数据集（如 MNIST、CIFAR）进行特征提取，图像像素具有空间平稳性。而本课题的难点在于：物联网恶意流量表现为高动态、非结构化的一维时序统计特征。如何设计有效的一维网络架构与损失函数，克服流量特征的剧烈波动，使其在隐空间中聚集成具有高区分度、高稳定性的“特征原型”，是首要难点。

2. “双重轻量化”压缩与极端 Non-IID 下的检测精度平衡

物联网环境不仅要求通信轻量化，更要求本地计算轻量化。在对本地检测模型进行大幅度精简（甚至引入剪枝或量化）以适应低端传感器的同时，极易导致模型对复杂变种恶意流量的特征表达能力骤降。在各设备遭遇攻击类型极端不平衡（Non-IID）的情况下，如何在极度压缩模型参数量的同时，防止罕见攻击类别（如特定变种僵尸网络）的“全局原型”发生灾难性遗忘，维持高检测精度，是本课题需要反复调优的工程与理论难点。

3. 端云协同联邦仿真系统的工程落地与并发调度

将联邦原型学习的理论机制转化为可运行的软件系统，并构建完整的“端云协同”闭环是本课题的重大工程难点。在基于深度学习框架开发系统时，需要在一台物理机上虚拟仿真出多个分布式的物联网节点。如何设计高内聚低耦合的软件模块、实现多节点并发本地训练的高效内存调度、精准控制极端 Non-IID 数据流的独立分发，以及确保“特征原型”在模拟端云交互时的严格同步与状态更新，要求极高的软件架构设计能力，以避免复杂仿真环境下的时序错乱或内存溢出问题。

# 课题创新点：

1. 提出面向一维流量特征的“双重轻量化”联邦检测范式

原版 FedProto 仅通过交换原型解决了“通信轻量化”问题，但其本地依然依赖标准的重型视觉网络（如 ResNet）。本课题创新性地将模型压缩技术与原型学习深度结合，在端云通信传输低维原型的同时，在本地节点设计并部署极简的 1D-CNN 或 MLP 网络。实现了“通信开销+本地计算负载”的双重指数级下降，真正契合了极度受限的物联网底层环境。

2. 构建兼容异构物联网硬件的流量安全协同框架

传统的物联网入侵检测系统多要求各节点部署同构模型。本课题利用原型学习“架构解耦”的特性，首次在恶意流量检测场景中实现了对硬件异构的完全兼容。允许高端智能网关与低端传感器根据自身算力运行完全不同深度的检测网络，仅通过在隐空间对齐“流量特征原型”进行安全威胁情报的共享，打破了物联网硬件异构导致的防御协同壁垒。

# 三、文献综述（或调研报告）：

随着物联网技术的迅猛发展与大规模部署，其在智能家居、智慧农业、工业控制等领域的应用日益广泛。然而，由于物联网底层设备通常受限于计算、存储和电池资源，难以部署复杂的安全防护机制，这使得物联网网络极易遭受各类网络攻击，尤其是以 Mirai、BASHLITE 为代表的僵尸网络恶意流量攻击。为了有效应对这一严峻的安全挑战，学术界与工业界在物联网恶意流量检测、分布式隐私保护以及模型轻量化领域展开了广泛的研究。本综述将结合近年来的核心文献，从传统物联网恶意流量检测技术、联邦学习在物联网安全中的应用，以及轻量化联邦学习框架三个维度，结合物联网恶意流量检测任务中面临的通信瓶颈、数据隐私与设备异构等实际挑战，对国内外相关研究现状进行系统梳理与深度剖析。

# 1. 物联网恶意流量检测技术研究现状

早期的网络入侵检测系统主要依赖于基于签名的技术，即通过深度包检测匹配已知的攻击特征库。然而，面对物联网环境中海量的加密流量和层出不穷的未知变种攻击，基于签名的检测方法逐渐暴露出泛化能力差、滞后性强等缺陷。

近年来，以深度学习为代表的异常检测技术逐渐成为主流。这类方法通过学习正常流量与异常流量的统计特征进行分类。在此背景下，MEIDAN 等人[1]提出了一种基于网络流量统计特征的深度自编码器检测方法，并公开发布了著名的 N-BaIoT 数据集。该研究通过提取网络流量的包长、到达时间间隔等一维统计特征，证明了利用机器学习进行僵尸网络异常行为识别的有效性。随后，ROOPAK 等人[2]系统评估了多层感知机（MLP）、一维卷积神经网络（1D-CNN）以及长短期记忆网络（LSTM）在物联网入侵检测中的表现，实验指出 1D-CNN 在处理序列化的网络流量特征时，能够在保证较高检测精度的同时显著降低系统的推理延迟，这为物联网终端部署轻量化模型提供了实践依据。近期，ELNAKIB 等人[3]提出了一种名为 EIDM 的深度学习入侵检测模型，进一步优化了针对复杂物联网攻击的特征提取网络，提升了模型对多种攻击类型的综合识别率。

尽管集中式的深度学习在检测精度上取得了突破，但在物联网场景下，其固有缺陷日益凸显：所有边缘设备必须将海量的网络流量数据实时上传至云端数据中心进行集中训练。这不仅给物联网有限的通信带宽带来了极其沉重的负担，更重要的是，网络流量数据中蕴含着大量的用户隐私，集中式上传面临着严峻的数据泄露与隐私合规风险。

# 2. 联邦学习在物联网安全中的应用现状

为了打破“数据孤岛”并解决集中式训练带来的隐私泄漏问题，MCMAHAN 等人[4]于 2017 年首次提出了联邦学习这一革命性的分布式机器学习范式。其核心算法 FedAvg 遵循“数据不动模型动”的原则，各参与节点在本地利用私有数据训练模型，仅将更新后的模型参数上传至中心服务器进行聚合，从而在从根本上保护数据隐私的前提下，实现全局模型的联合优化。

鉴于联邦学习在隐私保护方面的天然优势，研究人员迅速将其引入安全与入侵检测领域。LI 等人[5]针对工业信息物理系统的入侵检测问题，提出了 DeepFed 框架。该框架利用联邦学习联合多个工业站点共同训练深度学习模型，不仅有效保护了各工厂的工业数据隐私，还通过引入加密机制防止了模型参数在传输过程中的泄露。在智慧农业领域，FRIHA 等人[6]提出了 FELIDS 框架。FELIDS 探讨了在农业物联网这种高度分布式的资源受限环境中，如何利用联邦深度神经网络构建分布式的入侵检测系统，并在多种物联网攻击数据集上验证了其在多类别攻击检测下的鲁棒性。

然而，上述研究多基于传统的联邦平均算法，在实际的物联网部署时暴露出明显的瓶颈：庞大的模型参数传输依然会给窄带物联网带来巨大的通信压力，且传统联邦学习要求所有参与节点必须具备完全相同的模型架构，这严重脱离了物联网设备算力参差不齐的客观现实。

# 3. 轻量化联邦学习与高效训练研究

针对传统联邦学习在物联网环境下面临的通信瓶颈与计算资源限制问题，如何实现联邦学习的“轻量化”与高效训练成为了近年来的研究热点。

在通信优化与模型压缩方面，PRAKASH 等人[7]提出了一种对物联网设备极其友好的通信高效联

邦学习框架。该研究创造性地将模型剪枝与量化技术联合应用于联邦学习的本地训练与参数上传环节。通过剔除神经网络中冗余的连接权重，并将浮点数参数量化为低位宽整数，该框架在几乎不损失模型准确率的前提下，大幅降低了本地物联网设备的计算内存占用和网络通信开销。在高效训练机制方面，WANG 等人[8]提出了 Fednlr 框架，通过引入神经元级别的自适应学习率机制，有效加速了联邦学习的收敛过程并降低了无效的通信轮数。在国内学者的最新研究中，王鑫澳等人[9]针对参数量庞大的预训练模型提出了基于联邦学习的高效训练框架，其提出的基于稀疏化梯度更新与自适应压缩机制，为在通信受限网络中优化复杂深度模型提供了极具价值的参考思路，其压缩理念可有效迁移至物联网流量检测模型的轻量化设计中。

# 4. 数据与设备异构联邦学习及原型学习范式

物联网场景中最棘手的理论难题在于“统计异构性”（即 Non-IID 数据分布）与“系统异构性”（即硬件与模型架构不统一）。不同物联网设备由于所处网络位置不同，遭受的恶意流量攻击类型分布极不平衡；同时，不同设备的算力天差地别。

为了解决数据 Non-IID 问题，ZHOU 等人[10]深入剖析了异构数据下联邦学习模型平均机制失效的根本原因，并提出了一系列改进的模型聚合策略，以缓解数据分布不均导致的全局模型偏移。进一步地，DAI 等人[11]提出利用类别原型来应对联邦学习中的数据异构性。该研究指出，通过在特征空间中对齐各类别的数据原型，可以有效提升模型在极端 Non-IID 数据下的泛化能力，这为利用“特征原型”代替“模型参数”进行知识共享打开了思路。

为了彻底打破模型系统异构性的束缚，TAN 等人[12]提出了具有里程碑意义的联邦原型学习架构。FedProto 彻底颠覆了传统联邦学习交换“模型参数”的范式。在该框架下，各客户端利用本地模型对数据进行特征提取，计算各类样本在隐空间中的特征均值（即“特征原型”），并仅向服务器上传这些低维度的特征原型。

服务器对各节点上传的原型进行聚合计算出全局原型，再下发给客户端以指导本地模型的表征对齐。FedProto 将通信开销从庞大的模型参数量骤降至几十个维度的原型向量，实现了极致的通信轻量化。更重要的是，它实现了模型架构的解耦，允许高性能网关运行深层复杂网络，而低功耗传感器运行浅层简单网络，完美契合了物联网的硬件异构生态。

# 5. 文献总结与本课题切入点

综上所述，当前学术界在物联网恶意流量检测、联邦学习隐私保护以及轻量化优化等单一领域均取得了丰硕的成果。然而，综合考量物联网“隐私敏感、资源极度受限、硬件高度异构、数据高度Non-IID”的复杂特征，现有的安全防护体系仍需进一步融合创新。传统的 FedAvg 等算法虽解决了隐私问题，却难以在低算力、窄带宽的异构物联网中落地；而现有的轻量化入侵检测模型多局限于本地单机训练，缺乏全网威胁情报的协同共享。

基于上述研究现状与现实痛点，本课题致力于探索一条融合创新的技术路线。课题将以N-BaIoT[1] 等真实物联网流量数据集为基础，在客户端设计极简的轻量化特征提取模型（如 1D-CNN或 MLP）[2]；同时引入先进的 FedProto[12]框架替代传统的参数聚合机制。通过“本地轻量化模型$^ +$ 端云原型通信”的双重设计，并在必要时结合剪枝与量化技术[7]，本课题旨在理论和工程上解决物联网恶意流量检测中“数据隐私、通信过载、硬件异构”的冲突难题，构建一个高效、安全且普适的物联网恶意流量检测系统。

# 四、方案（设计方案、或研究方案、研制方案）论证：

根据任务书的要求，本课题旨在解决物联网环境中设备算力严重受限、联邦学习通信参数量过大、以及底层流量数据隐私极其敏感等核心痛点。为严格落实任务书中规定的三项核心子任务，本课题摒弃了传统的集中式深度学习与传统的参数聚合联邦学习架构，创新性地设计并实现一套基于轻量化联邦原型学习的物联网恶意流量检测仿真系统。

本方案的论证与设计紧密围绕端、边、云三层协同防御体系展开，并将工程落地点严格聚焦于基于 Python 与 PyTorch 框架的单机多节点分布式仿真实现。该方案既保证了理论层面的前沿性与严谨性，又确保了在规定周期内系统源码的可落地性。具体设计方案从以下四个核心维度进行详细论证：

# （一） 总体三层协同防御仿真架构设计

为高度契合物联网的分布式与异构特性，本系统在逻辑架构上设计为三层协同体系，并在仿真代码中进行面向对象的映射实现。系统的双重轻量化运行架构与核心端云数据流转如图 1 所示。

# 1. 感知层客户端仿真设计

感知层对应实际场景中海量且分布广泛的物联网终端设备，例如底层温湿度传感器或边缘智能网关。在本方案的单机仿真系统中，将通过代码实例化多个客户端对象来模拟这些分布式节点。每个节点实例将独立加载本地的非独立同分布网络流量数据，并根据预先设定的设备算力标签，部署不同复杂度的轻量化神经网络进行本地流量特征的提取与异常分类前向传播。

# 2. 联邦原型传输方案设计实现

这是本方案突破传统框架瓶颈的核心创新点。传统联邦学习必须在网络中传输海量的模型参数梯度，极易耗尽物联网带宽。本系统的端云交互打破了这一壁垒，仅通过联邦原型学习协议传输非敏感的特征原型。在代码层面，通信内容仅仅是传递维度极低的特征均值张量。这种机制不仅使通信开销呈指数级下降，更确保了原始流量数据和本地模型内部权重绝对不出本地，从根本上阻断了隐私泄露的链路。

# 3. 聚合层中央调度服务端设计

聚合层对应中央云服务器或高级聚合路由。在仿真系统中，表现为全局的服务器调度模块。其核

心职责是接收各边缘节点按通信批次上传的局部特征知识，在全局层面执行基于数据样本量的加权聚合算法，并将生成的全局统一原型广播下发，以此指挥并约束各本地边缘模型的下一轮更新。

# （二） 数据处理与轻量化检测模型设计

针对任务书“设计轻量化恶意流量检测模型，对网络流量数据进行特征提取”的明确要求，本方案在数据流水线与本地模型架构上的具体落地细节如下。

# 1. 流量特征工程与异构数据预处理

在实验数据的选择上，系统将采用公开的真实物联网恶意流量数据集，以此替代复杂的底层网卡实时抓包开发，确保工程的安全可控与高度可复现性。系统首先读取数据集中的数值型流量统计特征，例如网络流的五元组信息、包长均值与方差、网络流持续时间以及数据包到达时间间隔等。由于不同维度的特征量纲差异巨大，系统在构建数据加载器时，将强制引入归一化处理策略，将所有输入特征严格缩放至统一的区间内，随后将其重塑为一维张量，以作为后续轻量化一维卷积网络的标准输入格式。为了真实模拟物联网设备遭遇攻击类型极其不平衡的现状，系统还将采用狄利克雷分布算法对全局数据集进行强制切分，人为制造各客户端之间数据的非独立同分布特性。

# 2. 异构兼容的轻量化特征提取网络设计

传统的联邦学习算法强制要求所有客户端必须运行完全相同架构的模型。本方案充分利用原型学习算法架构无关的优越特性，允许不同算力的设备运行不同深度的网络，仅需在代码层面保证所有网络的最后一层输出维度严格对齐。对于算力较高的智能网关节点，系统将部署轻量化的一维卷积神经网络，通过一维卷积层提取时序流量序列的局部关联特征，配合批归一化层加速收敛，并通过最大池化层进行特征降维。对于算力极低的底层传感器节点，系统仅部署包含两到三层全连接层的多层感知机，极大降低前向传播的计算复杂度与内存占用。此外，在本地模型的定义模块中，系统还将预留网络剪枝或量化技术的接口，通过剔除网络中冗余的连接权重，进一步降低边缘设备的推理负载。

# （三） 基于原型学习的高效联邦框架设计

针对任务书“设计高效联邦学习框架，保证物联网流量数据隐私，高效训练恶意流量检测模型”的要求，本方案摒弃了高开销的梯度聚合模式，具体演进逻辑论证如下。

# 1. 客户端本地特征原型提取机制

在本地联邦训练周期的前向传播阶段，客户端针对特定的网络流量类别（例如某种特定的分布式拒绝服务攻击或正常流量），提取所有属于该类别样本的隐空间特征表示。随后，系统在代码中计算这些同类别特征向量的算术平均值，该均值向量即被定义为该客户端在该类别上的本地特征原型。在通信交互阶段，客户端仅将该低维度的原型向量上传至服务器。这一机制将单次通信的数据维度从数百万压缩至数十或数百维，实现了极致的轻量化。同时，由于上传的仅仅是高度抽象的类均值统计特征，攻击者无法通过逆向工程还原出原始的网络流量，从而提供了严密的隐私保护屏障。

# 2. 服务端异构原型加权聚合策略

服务器在接收到多个客户端上传的本地原型数据后，会根据各客户端对应类别的原始样本数据量规模，进行全局的加权平均计算，从而生成具有全局视野的特征原型。该聚合机制的计算复杂度极低，且完全屏蔽了底层各物联网设备的神经网络架构差异，从根源上解决了物联网硬件设备异构协同的理论难题。

# 3. 引入原型对齐正则化的本地模型更新

为了解决不同设备数据缺失带来的泛化能力差的问题，服务端会将聚合后的全局特征原型下发给各个客户端。在下一轮的本地训练中，客户端会在其原有的分类损失函数基础之上，强制引入一个基于均方误差的原型对齐惩罚项。该惩罚项专门用于计算本地网络提取的特征向量与全局中心原型之间的空间距离。通过在反向传播中最小化这一联合损失，系统在数学逻辑上强制要求本地设备提取的未知流量特征主动向全局公认的特征中心靠拢。这种方式使得各个设备在绝对不共享底层原始流量数据的前提下，实现了跨节点的高效知识蒸馏与威胁情报转移。

# （四） 仿真系统闭环实现与性能验证方案

本课题的最终工程成果体现为一套基于 Python 的联邦学习仿真评测系统源码。为实现“在物联网场景下进行性能验证，评估有效性”的终极目标，系统的代码流转与量化评估方案设计如下。

# 1. 单机仿真系统执行闭环架构

仿真系统源码将严格遵循模块化与高内聚的软件工程设计原则。在系统主入口程序中，将构建全局时序循环模块，以精细模拟端云交互的完整生命周期。该闭环流程依次为：触发客户端本地多轮次迭代训练、抽取网络隐层特征并计算本地类原型、模拟上行网络通信返回原型数据、服务端调用全局聚合算子生成新原型、模拟下行网络通信广播新原型、客户端载入全局原型并计算包含正则化惩罚项的联合损失以更新权重。该系统框架逻辑严密，完全契合毕业设计的系统级工作量要求。

# 2. 多维度量化评估指标体系设计

系统在每一轮联邦通信结束后，将在预留的全局测试集上计算并持久化输出核心验证指标。在安全检测效能方面，系统将基于混淆矩阵计算并输出全局准确率、精确率、召回率及综合评价指标，重点评估系统对各类变种恶意流量的漏报与误报情况。在核心的轻量化与开销对比方面，系统将引入传统的参数聚合联邦学习算法以及本地孤立训练模式作为基线对照组。仿真代码将实时统计并对比各方案的累计网络通信传输量、各异构节点的模型参数规模、单次前向推理的浮点运算次数，以及单条网络流量的端到端检测推理延迟。通过上述详实、多维的量化数据对比与消融实验，本方案将具备充足的客观依据来论证：该系统能够在大幅度降低物联网通信负担与边缘设备算力能耗的前提下，实现对海量恶意流量的精准识别与高效协同拦截。
