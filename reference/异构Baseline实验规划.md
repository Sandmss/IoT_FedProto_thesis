# 异构 Baseline 实验规划

记录时间：2026-05-12

## 一、规划目标

针对老师提出的“需要补充异构 baseline，并增加异构对比实验”的意见，本文拟在现有物联网恶意流量检测任务上，引入三种具有代表性的异构联邦学习方法作为 baseline，分别为 `FedProto`、`FD(FedDistill)` 和 `LG-FedAvg`。三者分别代表了原型聚合、知识蒸馏和局部-全局参数分离三种不同技术路线，能够较为系统地支撑模型异构场景下的对比分析。

本规划的核心目标包括：

1. 验证不同异构联邦方法在物联网恶意流量检测任务中的可行性。
2. 比较不同方法在模型异构条件下的分类性能与收敛表现。
3. 比较不同方法在异构程度变化下的稳定性与适应性。
4. 从通信开销和实现复杂度两个角度分析不同方法的适用性。

## 二、三类 Baseline 的异构理论基础

### 1. FedProto 的异构理论基础

`FedProto` 的核心思想是以“类别原型”替代“模型参数”作为联邦通信对象。类别原型本质上是某一类别样本在特征空间中的均值表示。对于第 `i` 个客户端，其本地模型先提取特征表示，再对同类样本特征求均值，得到局部类别原型；服务器对来自不同客户端的同类原型进行聚合，形成全局类别原型，并将其下发至客户端，用于约束本地训练过程。

这一机制的理论基础在于：虽然不同客户端的模型结构可以不同，但只要它们能够将输入样本映射到某个共享语义的表征空间，就可以通过类别级别的抽象原型进行知识交互，而不必要求模型参数具有相同维度和相同结构。因此，`FedProto` 天然适用于模型异构场景。

从异构联邦学习角度看，`FedProto` 具有以下理论特点：

1. 不依赖参数同构性。客户端之间不需要共享相同网络结构，因此适合 `MLP`、`CNN`、`Transformer` 等异构模型并存的场景。
2. 通过类别语义对齐缓解异构影响。虽然不同模型提取特征的方式不同，但同一类别的样本仍可在抽象层面进行原型对齐。
3. 通信负担较轻。相比传输全量模型参数，原型向量维度更低，更适合资源受限的物联网环境。

因此，`FedProto` 可以作为本文异构实验中最基础、最直接的 baseline，用于验证“仅通过原型交互是否足以支持异构客户端协同训练”。

### 2. FD(FedDistill) 的异构理论基础

`FD(FedDistill)` 属于基于知识蒸馏的异构联邦学习方法。其基本思想并不是聚合客户端模型参数，而是利用客户端输出的知识信息进行协同训练。通常而言，客户端会基于本地模型输出类别 logits、软标签或其他预测分布信息，服务器对这些知识进行整合后，再反馈给客户端作为蒸馏目标，用于指导后续训练。

该方法的理论基础在于：不同结构的模型虽然内部参数形式各不相同，但其最终输出空间通常具有一致的分类语义，因此可以在“输出分布”层面进行知识迁移。换言之，即使客户端使用的是不同的网络架构，只要它们面向同一分类任务，就可以通过软标签或 logits 的蒸馏机制共享知识，而无须直接共享模型本身。

从异构联邦学习角度看，`FD(FedDistill)` 具有以下理论特点：

1. 通过输出层知识而非参数进行协同，避免了不同模型结构之间无法直接平均参数的问题。
2. 蒸馏信息具有较强的任务语义性，可以反映类别间的相对相似性，因此对异构模型具有一定兼容性。
3. 方法范式与原型聚合不同，可作为 `FedProto` 的异路线对比对象，增强实验说服力。

因此，`FD(FedDistill)` 可作为本文的第二条异构 baseline，用于比较“原型级知识共享”与“输出级知识蒸馏”在物联网恶意流量检测任务中的异同。

### 3. LG-FedAvg 的异构理论基础

`LG-FedAvg` 的基本思想是在联邦训练过程中，将模型划分为“全局共享部分”和“本地专属部分”。其中，全局部分参与联邦聚合，本地部分保留在各客户端独立更新。其核心假设是：不同客户端虽然存在数据分布差异和任务偏好差异，但仍然可以共享一部分通用表示，而另一部分则应保留个性化能力。

该方法的理论基础在于分层表征学习思想，即神经网络的不同层承担不同功能：浅层或某些中间层往往学习较通用的表示，而高层或特定输出部分更偏向客户端本地任务特征。通过将共享参数限制在某一部分层中，可以在保留联邦协同能力的同时，提高对客户端异质性的适应能力。

从异构联邦学习角度看，`LG-FedAvg` 具有以下理论特点：

1. 通过“共享部分参数 + 保留本地部分参数”的方式，在同构与完全异构之间提供折中方案。
2. 相比完全参数聚合方法，它更能适应客户端间的差异性。
3. 相比 `FedProto` 和 `FD`，它仍然属于参数共享范式，因此能够提供不同类别的对比视角。

需要注意的是，`LG-FedAvg` 更适合具有部分可对齐结构的模型组合，因此在本文实验中，其实现重点不在于完全复制原论文设置，而在于探索其在 `MLP`、`CNN` 等轻量模型组合下的可用性。

## 三、三条 Baseline 的选择依据

本文最终选择 `FedProto + FD(FedDistill) + LG-FedAvg` 作为异构 baseline，主要基于以下考虑：

1. 三者技术路线差异明显，分别对应原型聚合、知识蒸馏和分层参数共享，能够构成较完整的异构联邦对比体系。
2. `FedProto` 与本文当前工作联系最紧密，可作为最直接的基础对照方法。
3. `FD(FedDistill)` 能补充不同范式的异构协同机制，避免实验只在原型方法内部比较。
4. `LG-FedAvg` 能从参数共享角度提供补充，对比“部分参数聚合”和“非参数聚合”路线的差异。

## 四、实验目标

异构 baseline 实验拟重点回答以下几个问题：

1. 在物联网恶意流量检测任务中，异构客户端之间是否能够通过不同联邦机制实现有效协同训练。
2. 在 `FedProto`、`FD(FedDistill)` 和 `LG-FedAvg)` 三种方法中，哪一种在当前任务上表现更优。
3. 随着异构程度变化，不同方法的性能是否稳定，是否存在明显退化。
4. 不同方法在通信效率和工程实现复杂度上有何差异。

## 五、实验分组设计

### 1. 基础异构实验

先以两类模型构建基础异构场景：

1. `MLP + CNN`
2. 客户端比例设置为 `5:5`

该实验的目标是优先验证三种 baseline 是否都能在当前项目中顺利运行，并形成一组基础可比结果。

### 2. 异构比例实验

在 `MLP + CNN` 组合基础上，进一步调整不同类型客户端比例：

1. `MLP:CNN = 5:5`
2. `MLP:CNN = 4:6`
3. `MLP:CNN = 3:7`

该实验用于分析不同异构程度配置下，三种方法的性能变化趋势，从而回应老师关于“比较不同异构客户端比例”的意见。

### 3. 扩展异构实验

若时间允许，可进一步加入第三类模型：

1. `MLP + CNN + Transformer`
2. 比例可设置为 `3:3:4` 或平均分配

该部分可作为扩展实验，用于展示方法在更复杂异构场景下的适用性，但不必作为论文主实验的唯一重点。

## 六、评价指标设计

结合老师的反馈，异构 baseline 实验不再重点强调 `AUC`，而应更多围绕分类性能、收敛效率和通信开销展开。

### 1. 分类性能指标

1. `Accuracy`
2. `Precision`
3. `Recall`
4. `F1-score`

其中，`F1-score` 在恶意流量检测任务中具有较强代表性，应作为重点指标之一。

### 2. 收敛表现指标

1. 达到最佳性能所需通信轮数
2. 固定通信轮数下的最终性能
3. 收敛曲线变化趋势

这些指标可用于说明不同方法的训练稳定性和收敛速度。

### 3. 通信开销指标

1. 单轮通信量
2. 总通信量
3. 达到目标性能所需累计通信量
4. 达到收敛时的累计传输成本

相比“每轮通信参数量”这一单一指标，上述指标更能真实反映不同方法在实际场景中的通信效率。

## 七、实施顺序

为降低实现风险，建议按照由易到难的顺序推进：

### 第一阶段：FedProto

1. 整理现有 `FedProto` 异构实验代码。
2. 优先跑通 `MLP + CNN`、比例 `5:5` 的基础实验。
3. 输出基础分类指标和通信指标。

### 第二阶段：FD(FedDistill)

1. 增加蒸馏式联邦训练流程。
2. 重点解决不同客户端模型输出知识的对齐问题。
3. 在与 `FedProto` 相同的异构配置下完成对比实验。

### 第三阶段：LG-FedAvg

1. 设计可共享层与本地层的划分方案。
2. 在尽量保持结构兼容的前提下完成基础对比。
3. 若 `MLP` 与 `CNN` 的共享困难较大，可优先尝试统一后的高层表示或分类头部分共享。

## 八、论文中的写作建议

在论文中，可以将该部分设置为“异构对比实验设计”或“模型异构场景下的对比实验”小节，内容建议按以下结构展开：

1. 异构实验背景与动机
2. 三种 baseline 的理论依据与选择理由
3. 异构客户端模型与比例设置
4. 分类性能对比结果
5. 收敛与通信开销分析
6. 不同方法适用性的讨论

## 九、预期结论

基于三种 baseline 的异构实验，预期可形成以下几类结论：

1. `FedProto` 在异构场景下具有较好的结构适配能力和较低通信负担。
2. `FD(FedDistill)` 能够在不同模型之间实现知识迁移，但其蒸馏效果可能受到输出表达能力限制。
3. `LG-FedAvg` 能在一定程度上兼顾全局共享与本地个性化，但对模型结构兼容性要求相对更高。
4. 若本文方法最终优于三条 baseline，则可更有力地说明其在物联网恶意流量检测异构场景下的有效性与优势。

## 十、最小可执行方案

若考虑时间和工程复杂度，建议优先完成如下最小实验闭环：

1. 模型组合：`MLP + CNN`
2. 客户端比例：`5:5`、`4:6`、`3:7`
3. 对比方法：`FedProto`、`FD(FedDistill)`、`LG-FedAvg`
4. 核心指标：`Accuracy`、`F1-score`、通信轮数、总通信量

完成上述内容后，即可形成一组较完整的异构 baseline 实验结果，为论文中“模型异构对比实验”部分提供直接支撑。
