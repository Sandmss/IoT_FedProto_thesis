# IoT_FedProto 项目与开题报告对照差距清单

## 三、与开题报告相比仍缺失的关键内容

### 1) “异构客户端”尚未真正实现（高优先级）
开题报告强调不同算力设备运行不同模型（如网关与传感器异构），但当前 `main.py` 通过单一 `model_family` 配置，实际训练通常是全客户端同构。

缺口表现：
- 缺少“按客户端分配模型架构”的机制（如部分客户端 MLP、部分客户端 CNN1D）。
- 缺少异构场景下对比实验结果。

### 2) 论文级评估指标不完整（高优先级）
当前主要输出 Acc/AUC，尚不足以支撑“类别不平衡与安全检测任务”的论证强度。

缺口表现：
- 缺少 Macro-F1。
- 缺少 Precision / Recall（至少宏平均或按类）。
- 缺少混淆矩阵与按类指标持久化输出。

### 3) “双重轻量化”量化证据不足（高优先级）
开题报告强调通信与计算双轻量化，但现有代码未形成完整统计闭环。

缺口表现：
- 缺少通信开销统计（每轮上行/下行原型字节数、累计传输量）。
- 缺少模型规模统计（参数量）与推理时延/吞吐/FLOPs 对比。
- 缺少与基线在“开销-精度”维度的系统化对照表。

### 4) 剪枝/量化仍停留在“计划层面”（中高优先级）
报告提及可引入剪枝或量化，但当前尚无可开关、可复现实验入口。

缺口表现：
- 无剪枝/量化训练或推理路径。
- 无“压缩率-精度变化”结果。

### 5) FedProto 收尾功能存在未完成风险（中优先级）
`serverproto.py` 在训练结束处调用了可视化函数，但仓库中相关实现不完整，存在运行到末尾报错的风险。

缺口表现：
- 训练闭环稳定性可能受影响。
- 可视化产物与论文图表链路不稳定。

### 6) 实验矩阵与消融设计未系统化（中优先级）
报告强调对照与消融，但当前缺少成体系的实验脚本和统一汇总输出。

缺口表现：
- 缺少统一实验配置管理（不同 alpha、feature_dim、客户端异构比例等）。
- 缺少自动汇总报表（表格/图）用于论文直接引用。

## 四、建议优先级与下一步执行顺序

### P0（先做）
1. 修复并确认 FedProto 全流程稳定收尾（避免可视化调用导致训练末端失败）。
2. 补齐评估指标：Macro-F1 + Precision/Recall + 混淆矩阵保存。
3. 增加异构客户端策略：按客户端 ID 或配置分配 `MLP_IoT/CNN1D_IoT`。

### P1（随后）
4. 增加通信与计算开销统计（字节数、参数量、推理时延/FLOPs）。
5. 输出统一结果文件，形成“精度-开销”对照表。

### P2（论文增强）
6. 加入剪枝/量化实验入口与消融实验矩阵。
7. 固化实验脚本，形成可复现的一键批量实验流程。

当前系统已实现基于 FedProto 的端云协同训练主流程，完成了 IoT 一维流量特征建模、Dirichlet Non-IID 数据构造、客户端原型提取与服务端加权聚合等核心功能。
